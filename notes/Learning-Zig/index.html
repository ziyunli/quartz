    # Installation

    Zig is moving fast, so I do not recommend installing it from the package manager (e.g. Homebrew, nix, etc.) as they are usually lagged. Instead, follow what zigleanr suggests, download the latest master build from https://ziglang.org/download/ and put it in your $PATH. Note that you want to put the entire directory in your $PATH instead of the binary itself; otherwise you will get an error like this:

    1

    error: unable to find zig installation directory '/usr/local/bin/zig': FileNotFound

    # Strings

    Strings use double quotes. Zig stores strings as arrays of bytes.

    1

    var foo = "Hello, world!";

    Multi-line string literals are supported with leading \\ at the beginning of each line.

    1
2
3

    const foo =
  \\Hello,
  \\world!

2
3

  // ...

    # Switch

    switch works like match expression in other languages. Switch statements must be “exhaustive”. They are also valid expressions.

    1
2
3
4
5

    switch (players) {
  1 => print("1 player", .{}),
  2 => print("2 players", .{}),
  else => print("{} players", .{ players })
}

    unreachable is a special value that can be used to indicate that a switch statement is exhaustive.

    # Functions

    Zig functions are private by default but the main() function should be public. A function is declared public with the pub statement.

    # Defer

    defer is a statement that executes a function when the current scope is exited. errdefer is a statement that executes a function when the current scope is exited with an error.

    # Errors

    Errors are created in “error sets”, which works like an enum.

    1

    const Error = error{ Foo, Bar };

    Zig support “error unions”, which could be either a regular value OR an error from a set.

    1

    var text: MyErrorSet!Text = ...;

    !void will let Zig to infer the error type, which is useful for main().

    # Error handling

    catch catches an error and replace it with a default value.

    1

    foo = canFail() catch "bar"

    catch can also capture the error and perform additional actions.

    1
2
3
4
5

    foo = canFail() catch |err| {
  if (err == FishError.TunaMalfunction) {
    ...
  }
}

    try is a shorthand for catch to return the error.

    1
2
3
4
5

    canFail() catch |err| return err;

// is equivalent to

try canFail();

    if can be used to check if an error is present.

    1
2
3
4
5
6

    if (canFail()) |value| {
  // ...
} else |err| switch {
  FishError.TunaMalfunction => ...,
  else => ...,
}
